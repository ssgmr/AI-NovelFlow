# 分镜生成页面布局优化 - 提案

## 变更名称

`optimize-chapter-generate-page-layout`

## 变更类型

- [x] 新功能
- [ ] Bug 修复
- [x] 重构/优化
- [ ] 文档更新

## 变更概述

重构 ChapterGenerate 页面的整体布局和交互流程，从当前的 Tab 式离散操作改为**四阶段工作流 + 三栏式布局**，提升用户体验的连贯性和操作效率。

## 背景与动机

### 当前问题

1. **操作割裂**：用户需要在 4 个 Tab 之间频繁切换才能完成完整工作流
2. **上下文缺失**：每个 Tab 内的信息相互隔离，无法在同一视图看到相关资源
3. **JSON 编辑器束缚**：用户不关心 JSON 数据，但当前设计将 JSON 编辑器放在核心位置
4. **状态不透明**：ComfyUI 状态在右下角独立卡片，与生成流程脱节
5. **导航不便**：分镜切换依赖"上一个/下一个"按钮，没有全局导航视图

### 用户真实工作流

```
分镜拆分 → 音频生成 → 分镜图生成 → 视频生成
   ↓           ↓           ↓          ↓
 编辑描述    配置台词    生成图片    图转视频
 调整角色    上传音频    编辑提示词  添加转场
 设定场景                 调整资源    合并导出
```

## 变更目标

| 目标 | 当前状态 | 目标状态 |
|------|----------|----------|
| 流程连贯性 | 4 个离散 Tab | 4 个阶段式工作流 |
| 上下文完整性 | 信息分散 | 三栏布局，同屏可见 |
| 数据编辑方式 | JSON 编辑器 | 可视化表单编辑 |
| 系统状态展示 | 独立卡片 | 右侧边栏整合 |
| 分镜导航 | 前后按钮 | 底部缩略图导航条 |

### 包含内容

- ✅ 四阶段工作流设计（分镜拆分、音频生成、分镜图生成、视频生成）
- ✅ 三栏式布局实现（左侧资源栏 + 中间编辑区 + 右侧状态栏）

### 不包含内容

- ❌ 后端 API 修改（接口保持兼容）
- ❌ 业务逻辑修改（生成功能保持原有逻辑）
- ❌ ComfyUI 工作流修改

## 预期收益

1. **减少 Tab 切换次数**：从平均每任务 8-10 次降低到 2-3 次
2. **提升状态可见性**：ComfyUI 队列、任务进度实时可见
3. **降低认知负担**：JSON 隐藏，用户专注业务内容
4. **批量操作效率**：支持自定义选择分镜进行批量生成

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 布局复杂度高 | 开发周期长 | 分阶段实现，先核心后优化 |
| Store 重构风险 | 可能影响现有功能 | 保留原有 API，渐进式迁移 |
| 用户适应成本 | 老用户需要适应 | 保持操作逻辑一致，仅改变布局 |

> **状态**: 已完成

### 与原设计的差异

1. **Tab 顺序调整**: 实际实现为 分镜拆分(0) → 分镜图生成(1) → 音频生成(2) → 视频生成(3)
2. **组件结构简化**: Header、LeftPanel、RightPanel 采用内联实现
3. **音频生成 Tab 独立布局**: 不使用标准 ThreeColumnLayout
4. **侧边栏宽度调整**: 默认宽度 200px，右侧栏最大 450px
5. **底部导航条可折叠**: 新增折叠功能，展开高度 160px
