# 工作流系统规格说明

> **状态**: 已实现
> **最后更新**: 2026-03-08
> **注意**: 实际 Tab 顺序为 分镜拆分(0) → 分镜图生成(1) → 音频生成(2) → 视频生成(3)

## ADDED Requirements

### Requirement: 四阶段式工作流

系统应提供四阶段式工作流，引导用户完成从分镜拆分到视频生成的完整流程。

#### Scenario: 阶段顺序引导
- **WHEN** 用户进入 ChapterGenerate 页面
- **THEN** 系统应按顺序展示四个阶段：分镜拆分 → 分镜图生成 → 音频生成 → 视频生成

#### Scenario: 阶段依赖提示
- **WHEN** 前置阶段未完成时用户尝试访问后续阶段
- **THEN** 系统应友好提示用户需要先完成前置阶段

#### Scenario: 阶段进度跟踪
- **WHEN** 用户完成某个阶段的操作
- **THEN** 系统应记录该阶段的完成状态并在 Tab 上显示

---

### Requirement: 分镜拆分 Tab (阶段 0)

阶段 0 应支持分镜的拆分、编辑和管理。

#### Scenario: 原文内容展示
- **WHEN** 用户进入分镜拆分 Tab
- **THEN** 左侧区域应显示章节原文内容

#### Scenario: AI 智能拆分
- **WHEN** 用户点击"AI 拆分"按钮
- **THEN** 系统应调用 LLM 将章节内容智能拆分为多个分镜

#### Scenario: 分镜列表编辑
- **WHEN** 分镜拆分完成后
- **THEN** 用户应能编辑每个分镜的描述、调整分镜顺序

#### Scenario: 添加/删除分镜
- **WHEN** 用户需要调整分镜数量
- **THEN** 系统应支持添加新分镜和删除已有分镜

#### Scenario: 保存分镜数据
- **WHEN** 用户完成分镜编辑后点击保存
- **THEN** 系统应将分镜数据保存到后端，并标记阶段 1 为完成状态

---

### Requirement: 分镜图生成 Tab (阶段 1)

阶段 1 应支持分镜图的生成和管理。

#### Scenario: 角色列表展示
- **WHEN** 用户进入音频生成 Tab
- **THEN** 左侧区域应显示本章出现的所有角色列表

#### Scenario: 音色选择与试听
- **WHEN** 用户为角色选择音色
- **THEN** 用户应能试听所选音色

#### Scenario: 台词配置
- **WHEN** 用户查看分镜台词
- **THEN** 系统应显示该分镜中各角色的台词内容

#### Scenario: 批量生成音频
- **WHEN** 用户点击批量生成按钮
- **THEN** 系统应将为所有未生成音频的台词批量生成音频

#### Scenario: 任务进度显示
- **WHEN** 音频生成任务进行中
- **THEN** 系统应实时显示生成任务的进度状态

---

### Requirement: 音频生成 Tab (阶段 2)

阶段 2 应支持角色台词的音频生成。

> **注意**: 音频生成 Tab 使用独立的三栏布局，不使用标准的 ThreeColumnLayout。

#### Scenario: 资源面板展示
- **WHEN** 用户进入分镜图生成 Tab
- **THEN** 左侧资源面板应显示角色图、场景图、道具图列表

#### Scenario: 分镜大图预览
- **WHEN** 用户选择某个分镜
- **THEN** 中间区域应显示该分镜的大图预览和描述编辑区

#### Scenario: 智能优化提示词
- **WHEN** 用户点击"智能优化"按钮
- **THEN** 系统应使用 LLM 优化分镜描述为更适合图像生成的提示词

#### Scenario: 单张分镜图生成
- **WHEN** 用户点击生成单张分镜图
- **THEN** 系统应调用 ComfyUI 工作流生成该分镜的图片

#### Scenario: 批量分镜图生成
- **WHEN** 用户在批量模式下点击批量生成
- **THEN** 系统应依次为选中的多个分镜生成分镜图

#### Scenario: 资源重置/替换
- **WHEN** 用户需要更换角色图/场景图
- **THEN** 系统应支持重置或替换资源图中的角色/场景/道具

---

### Requirement: 视频生成 Tab (阶段 3)

阶段 3 应支持视频生成、转场添加和合并导出。

#### Scenario: 视频列表展示
- **WHEN** 用户进入视频生成 Tab
- **THEN** 左侧区域应显示所有分镜的视频列表

#### Scenario: 视频预览播放
- **WHEN** 用户选择某个视频
- **THEN** 中间区域应能播放该视频

#### Scenario: 转场生成
- **WHEN** 用户需要添加转场效果
- **THEN** 系统应支持生成两个分镜之间的转场视频

#### Scenario: 转场配置
- **WHEN** 用户配置转场参数
- **THEN** 系统应支持设置转场时长、转场效果类型

#### Scenario: 视频合并
- **WHEN** 用户点击合并视频
- **THEN** 系统应将所有分镜视频和转场视频合并为完整章节视频

#### Scenario: 视频导出
- **WHEN** 用户点击导出视频
- **THEN** 系统应导出合并后的视频文件供用户下载

---

### Requirement: 可视化表单编辑

系统应提供可视化表单编辑替代 JSON 编辑器。

#### Scenario: 分镜描述编辑
- **WHEN** 用户编辑分镜描述
- **THEN** 系统应提供文本域供用户输入和编辑分镜描述

#### Scenario: 角色多选
- **WHEN** 用户设置分镜中的角色
- **THEN** 系统应提供多选下拉框，支持搜索和选择多个角色

#### Scenario: 场景选择
- **WHEN** 用户设置分镜场景
- **THEN** 系统应提供单选下拉框，支持搜索和选择场景

#### Scenario: 道具多选
- **WHEN** 用户设置分镜中的道具
- **THEN** 系统应提供多选下拉框，支持搜索和选择多个道具

#### Scenario: 时长设置
- **WHEN** 用户设置分镜时长
- **THEN** 系统应提供数字输入框供用户设置分镜时长 (秒)

#### Scenario: 表单值同步
- **WHEN** 用户修改表单中的任何字段
- **THEN** 系统应实时更新 Store 中的数据状态
